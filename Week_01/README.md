## 心得

1. 先思考解法，把想到的解法列举出来->一种思路：最近重复子问题（复杂性低）
2. 思路清晰后再开始敲代码，否则往往漏洞百出
3. 先易后难：先做最简单的解法（C往往超时💔）
4. 自顶向下：先写出脉络结构，再补充细节
5. 优化：升维；空间换时间
6. 如果懵逼了：暴力?->基本情况?->找最近重复子问题(泛化)
